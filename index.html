Prototype Writeup

What have I done so far?

I created an initial public-facing webpage for the Steam games project and connected it to the provided Steam CSV dataset. The page loads the data, computes genre-level summaries, and displays an interactive bar chart that can switch between game count, average rating, average playtime, and average price. I also added a genre filter so a viewer can focus on a single first-listed genre and compare that subset against the full dataset summary. The current prototype includes a GitHub repository link, a working visualization, hover details, metric controls, and a written project description so it satisfies the core initial prototype requirements.

What will be the most challenging part to design and why?

The hardest part will be designing a visualization that shows meaningful differences without letting the largest genres overwhelm the smaller ones. Steam games can belong to multiple genres and tags, so choosing whether to use first-listed genre, all listed genres, or SteamSpy tags will strongly affect the story the visualization tells. It will also be challenging to balance several variables at once, because price, positive rating, number of owners, and playtime all have skewed distributions with major outliers. The final design will need careful interaction choices so users can explore the dataset freely while still understanding the main takeaway quickly.
